
## Восстановление зависимости нейронной сетью

Прежде, чем рассматривать процесс обучения нейронной сети, мы рассмотрим понятие **размеченной обучающей выборки**.

## Что такое размеченная обучающая выборка?
Размеченная обучающая выборка состоит из некоторого количества объектов, и для этих объектов мы знаем две вещи:

1. **Признаки**: для каждого объекта $i$ известен набор признаков $x_i = (x_{i1}, x_{i2}, \dots, x_{iD})$, где $D$ — размерность признакового пространства.
2. **Метка объекта**: для каждого объекта $i$ известна соответствующая метка $y_i$.

Таким образом, выборка имеет вид:

$$
\{(x_1, y_1), (x_2, y_2), \dots, (x_N, y_N)\}
$$

где $N$ — количество объектов в выборке.

---

### Пример: зависимость силы ветра от дня

**Признаки дня (вектор $x_i$):**
- Порядковый номер дня в году.
- Наличие облачности (0 — нет облаков, 1 — облачно).
- Температура в этот день.
- Атмосферное давление.
- и другие метеорологические параметры.

**Метка ($y_i$):**
- Скорость ветра в метрах в секунду.

То есть по набору признаков дня мы хотим восстановить (предсказать) силу ветра в этот день.

---

## Разделение выборки на подмножества
Прежде чем обучать модель, принято разделять исходную выборку на три части:

1. **Тренировочный датасет (training dataset):**
   - Используется **непосредственно для обучения** модели.
   - На этих данных модель "видит" примеры и подстраивает свои параметры.

2. **Валидационный датасет (validation dataset):**
   - Используется для **подстройки гиперпараметров** модели.
   - На этом датасете оценивается качество модели в процессе обучения, но он **не используется для обновления весов**.
   - Помогает избежать переобучения и выбрать лучшую конфигурацию модели.

3. **Тестовый датасет (test dataset):**
   - Используется **только для финальной оценки** качества модели.
   - Этот датасет модель **никогда не видит** в процессе обучения и настройки гиперпараметров.
   - Хороший результат на тестовом датасете свидетельствует о том, что модель **успешно обобщила** информацию из тренировочных данных.

## Математическая модель задачи восстановления
Рассмотрим задачу восстановления скорости ветра по параметрам дня.

Предположим, существует **скрытая зависимость** $f(x)$, где:
- $x$ — вектор признаков (например, давление, температура и др.),
- $y$ — целевая переменная (скорость ветра).

Однако реальные измерения содержат **погрешности**, поэтому наблюдаемые значения $y_i$ описываются как:

$$
y_i = f(x_i) + \varepsilon_i
$$

где:
- $f(x_i)$ — истинное (скрытое) значение скорости ветра при признаках $x_i$,
- $\varepsilon_i$ — погрешность измерения (шум), которая может быть как случайной, так и систематической.

**Наша цель:** по наблюдениям $\{(x_i, y_i)\}_{i=1}^N$ восстановить (аппроксимировать) скрытую функцию $f$.
